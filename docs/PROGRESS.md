## 現在のブロッカー / 要判断

1. **Phase 3b（カメラ実色）確認待ち**: ボクセルが実物の色になるか、色の向き(上下/左右)が合うかを確認してほしい。ズレていれば「🔃 色向き」で合わせられる。全く色が出ない場合は readback 非対応の可能性 → 別方式(three ExternalTexture)に切替。
2. **キャッシュ対策**: 画面に `build:` 時刻を表示。古い版を見てしまったら `?v=<番号>` を付けて開くと最新が読める（GitHub Pages は index.html を ~10分キャッシュするため）。

## Phase 3b 実機確認手順（GitHub Pages — 開発機不要）

1. マージ → Actions 緑 → Pixel で **`https://kyoujyu88.github.io/VoxelWorld/?v=6`**（`?v=` でキャッシュ回避、番号は任意）
2. 画面下の `build:` 時刻が新しいことを確認（古ければ番号を変えて再読込）
3. 「開始」→ 部屋を見回す。ボクセルが**実物の色**になるか確認:
   - 色が合っていれば OK。上下/左右が反転していたら「🔃 色向き」を数回押して合わせる（合った時の「色向き X/Y」を教えて）
   - 「🎨 色: カメラ/高さ」で切替（高さ = 青緑赤のフォールバック）
   - 右のサムネイルにカメラ縮小画像が出れば readback 成功
4. スクショ＋「実色になった？向きは合ってる？」を返してほしい

> 色が全く出ない / サムネイルが真っ黒なら readback 非対応の可能性。その旨を教えてくれれば別方式に切替える。開発機がある場合は `docs/ENVIRONMENT.md` の方法 B でも確認できる。
