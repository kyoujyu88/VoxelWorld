## 現在のブロッカー / 要判断

1. **スキャンノイズ = 本質課題（Phase 6.3 で minObs スライダー化・実機で調整中）**: 実機所感「カービングで浮きは一応消えるが、スキャン精度自体が低く、別角度に動くと中に浮くボクセルが結構出る」。原因は低解像度深度(160×90)＋わずかな姿勢ドリフトで、同じ面が角度ごとに少しずれたセルに入り**面の周りに厚いノイズ殻**ができること。カービング(8cm マージン)は面の手前の浮きは消すが、面近傍の殻は残る。→ **`minObs`(描画に必要な観測数)を 3→6 に上げ、かつ実機で回せる「スキャン安定性」スライダー(3〜14)を追加**。高いほど疎なノイズ殻が落ち密な真の面が残る＝クリーン。副次的に描画数が減り FPS も回復(充填 0.9 のオーバードローで 24〜32fps に低下していた)。実機で最適値を探る。
2. **カービング（Phase 6.2・一部実機確認）**: 浮きは「一応消える」。効き具合/マージンは minObs と合わせて調整。**既知の限界**: 俯瞰プレビューは削除を反映しない(暫定)。2cm ライブ色は最初の色で固定。
3. **超広域の範囲（Phase 6.1 で 200 万まで拡大済み・実機で 966k まで確認）**: さらに広い場が必要なら SoA 化(省メモリ)か走査解像度可変を検討。
4. **キャッシュ対策**: 画面に `build:` 時刻を表示。古い版を見てしまったら `?v=<番号>` を付けて開くと最新が読める（GitHub Pages は index.html を ~10分キャッシュするため）。

## Phase 6.1 実機確認記録（2026-07-23 / `build: 21:13Z`）

- **範囲拡大 OK**: `ボクセル(2cm) 966,022 セル`（旧 60 万上限を大きく超過）で `FPS 49`・**非クラッシュ**。→ 範囲拡大は成功。
- **色の重み付けは体感で分かりにくい**（2cm ライブは最初の色で固定のため）。→ 本質でないので後回し。
- **本質的な指摘（→ Phase 6.2）**: 何もない空間の**浮きボクセルが近づいても残り、裏の壁/物を隠す**。近づいたら消してほしい。また「マイクラのようにグリッド状の塊」が最終イメージ（内部は元々 2cm グリッド整列。見た目が散らばるのは充填率 0.55 とノイズのため）。

## Phase 6.2 実機確認記録（2026-07-24 / `build: 02:11Z`）

- カービングで浮きは**一応消える**（`一応中に浮いているボクセルは消えてる`）。充填 0.9 で塊感は出た。
- **本質課題の特定**: スキャン精度が低く、別角度に動くと中に浮くボクセルが結構出る。`ボクセル 88〜92 万`・`描画 250,000/250,000`（上限飽和）・`FPS 24〜32`（充填 0.9 のオーバードローで低下）。→ ノイズ低減が必要 → Phase 6.3。

## Phase 6.3 実機確認手順（スキャン安定性スライダー — 開発機不要）

1. マージ → Actions 緑 → Pixel で `https://kyoujyu88.github.io/VoxelWorld/?v=14`（`?v=` でキャッシュ回避、番号は任意）
2. `build:` 時刻が **`02:11Z` より新しい**ことを確認
3. スキャン → 「表示サイズ」の下の **「スキャン安定性」スライダー**を右へ動かして離す（既定 6、範囲 3〜14）:
   - 右へ（値↑）で**浮きボクセル/ノイズ殻が減り、面がクリーンになる**か（`描画中` の数も減る＝FPS 回復）
   - 左へ（値↓）で拾いやすくなる代わりにノイズも増える。**ノイズが消えて実物の面が残る最適値**を探してほしい
   - 別角度に動いても中の浮きが増えにくくなったか
4. スクショ（安定性の値と `描画中`・`FPS` が見えるもの）＋「いくつが一番きれい？その時 FPS は？まだ浮く？」を返してほしい

> スライダーは既存データを再構成する（再スキャン不要）。最適値が分かればそれを既定にする。まだ不十分なら深度レンジ短縮/カービング強化/孤立ボクセル除去を追加検討。俯瞰プレビューは削除未反映（暫定）。
